# Experiment 3 Runner: Varying the Degree Scale

This README explains how to run **Experiment 3**, where the privacy-utility behavior is evaluated across different degree scales, represented by different effective \(\theta_0\) levels.


## 1. Required folder structure

Use the following flat folder layout:

```text
ProjectRoot/
  Utility/
    gapptr_utility.py
    Edge_flip.py
  Experiment3/
    run_exp3_vary_theta0_all_settings.py
    E3S1theta1/
      rep000/
      rep001/
      ...
    E3S1theta2/
      rep000/
      rep001/
      ...
    ...
    E3S1theta7/
      rep000/
      ...
    E3S2theta1/
      rep000/
      rep001/
      ...
    E3S2theta2/
      rep000/
      rep001/
      ...
    ...
    E3S2theta7/
      rep000/
      ...
```

The folder names are:

```text
E3S1theta1, E3S1theta2, ..., E3S1theta7
E3S2theta1, E3S2theta2, ..., E3S2theta7
```

Here `S1` means Setting 1 and `S2` means Setting 2.


## 2. Methods

Experiment 3 studies how the clustering error changes as the degree scale changes.

For both settings, the default total privacy budget is \(\varepsilon_{\mathrm{all}} = 0.8.\)
The runner evaluates:

| Method | Privacy budget |
|---|---|
| `GapPTR_oracle` | \(\varepsilon=0.8,\ \varepsilon_1=0\) |
| `GapPTR_tilde` | \(\varepsilon=0.6,\ \varepsilon_1=0.2\) |
| `EdgeFlip` | \(\varepsilon=0.8\) |
| `NonPrivate` | no privacy noise |

Thus `GapPTR_tilde` uses the split

\[
\varepsilon = \varepsilon_{\mathrm{all}} - \varepsilon_1 = 0.6,
\qquad
\varepsilon_1 = 0.2.
\]

## 3. Data settings and folder mappings

### Setting 1: regular heterogeneous degrees

This setting uses

\[
(p_{\mathrm{in}},p_{\mathrm{out}})=(0.4,0.1),
\qquad
\theta_i\sim \mathrm{Unif}(\theta_{\min},\theta_{\max}).
\]

The default folder-theta mapping is:

| Folder | \(\theta_0\) used in plot |
|---|---:|
| `E3S1theta1` | 0.10 |
| `E3S1theta2` | 0.15 |
| `E3S1theta3` | 0.20 |
| `E3S1theta4` | 0.25 |
| `E3S1theta5` | 0.30 |
| `E3S1theta6` | 0.35 |
| `E3S1theta7` | 0.40 |

### Setting 2: low-degree mixture

This setting uses

\[
(p_{\mathrm{in}},p_{\mathrm{out}})=(0.9,0.3).
\]

The degree parameters are generated by first drawing

\[
\theta_i\sim \mathrm{Unif}(\theta_{\min},\theta_{\max}),
\]

then randomly selecting 40% of nodes and multiplying their degree parameters by `0.1`:

```python
theta = rng.uniform(theta_min, theta_max, size=n)
mask = rng.random(n) < 0.4
theta[mask] *= 0.1
```

Equivalently, 60% of nodes have degree scale in
\([\theta_{\min},\theta_{\max}]\), while 40% have degree scale in
\([0.1\theta_{\min},0.1\theta_{\max}]\).

The default folder-theta mapping is:

| Folder | \(\theta_0\) used in plot |
|---|---:|
| `E3S2theta1` | 0.18 |
| `E3S2theta2` | 0.24 |
| `E3S2theta3` | 0.30 |
| `E3S2theta4` | 0.36 |
| `E3S2theta5` | 0.42 |
| `E3S2theta6` | 0.50 |
| `E3S2theta7` | 0.55 |

## 4. Running the experiment

From the `Experiment3/` folder, run:

```bash
python run_exp3_vary_theta0_all_settings.py
```

If each folder has only 5 replications, run:

```bash
python run_exp3_vary_theta0_all_settings.py --n-rep 5
```

Run only Setting 1:

```bash
python run_exp3_vary_theta0_all_settings.py --settings setting1 --n-rep 5
```

Run only Setting 2:

```bash
python run_exp3_vary_theta0_all_settings.py --settings setting2 --n-rep 5
```
